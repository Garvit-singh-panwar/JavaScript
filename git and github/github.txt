🧠 Creating a Repository on GitHub (Simple Notes)
🔹 How to Create a New Repository
Go to GitHub
Click on “Repositories”
Click the green “New” button
🔹 Fill the Details
Owner → Your account (or organization)
Repository Name → Give your project a name
Description (optional) → Short info about your project
🔹 Public vs Private
Public → Anyone can see your code 🌍
Private → Only selected people can see 🔒
🔹 README.md File
You can check ✔ “Add a README.md”

👉 What is README.md?

A file that explains your project
.md = Markdown (simple formatting language)

👉 What to write in it:

What your project does
Why you made it
Features
How to use it

👉 Good practice:
✔ Always create a README file

🔹 After Creating Repo
Your repository is created 🎉
You may see:
Branches (like main)
README.md file (if added)
1 commit (initial commit)
🔹 What is a Commit?
A commit = saved version of your code

👉 It shows:

What changes were made
When they were made
Who made them
🔹 Steps of Commit (Corrected)

You were close, just a small correction:

1. Add (Stage Changes)
You tell Git:
👉 “These changes are ready to be saved”

Example:

You created a new file
You edited some code
2. Commit (Save Changes)
You save those changes permanently
You must write a message (very important)

Example message:

"Added login page"
"Fixed bug in navbar"
🔹 Easy Example (Your Letter Example Improved ✉️)
Add → You wrote the letter and put it in envelope
Commit → You sealed it and wrote what’s inside
🔹 Editing Files on GitHub
Open a file
Click the ✏️ (pencil icon)
Make changes
Click “Commit changes”
🔹 Commit Window
You must write a commit message
Then choose:
Commit to main branch
OR create a new branch

👉 Then click “Commit changes”

🔹 What is a Branch?
A branch = separate version of your project

👉 Example:

main → original code
feature-login → new feature

👉 You can:

Work safely without affecting main code
Merge later


🎯 One-Line Revision
Repository = project folder
README.md = project explanation
Commit = saved change
Add = prepare changes
Branch = separate work
