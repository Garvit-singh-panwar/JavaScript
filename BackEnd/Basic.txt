## 1. The Big Picture: FrontEnd vs. BackEnd
Think of a restaurant. 
* **FrontEnd (The Dining Room):** The decor, the menu, and the tables. It’s what you see and interact with.
* **BackEnd (The Kitchen):** Where the cooking happens, the ingredients are stored, and the recipes (logic) are followed. You don’t see it, but without it, you don’t get your meal.



---

## 2. The Client-Server Model
In web development, we use a communication system called the **Client-Server Model**.
* **The Client:** Your device (phone or laptop) running a browser or an app.
* **The Server:** A powerful computer somewhere else that "serves" the data you need.

### How they talk: HTTP
To communicate, they use a language (protocol) called **HTTP** (HyperText Transfer Protocol). 
* The Client sends a **Request**.
* The Server sends a **Response**.

---

## 3. The Journey of a Click (Example: 20% Discount Link)
When you click a link in Facebook Messenger, a complex "handshake" happens behind the scenes:

1.  **DNS Resolution:** Your browser doesn't understand "facebook.com." It needs an **IP Address** (like a phone number for computers, e.g., `192.168.1.1`). It uses a "phonebook" called **DNS** to find that IP.
2.  **Establishing a Connection:** Once the browser has the IP, it uses **TCP** (Transmission Control Protocol) to make sure a stable connection is "plugged in" between you and the server.
3.  **Sending the Request:** Your browser sends an HTTP request to the server's **Port Number** (a specific "door" on the server designated for web traffic).



---

## 4. The "Verbs" of the BackEnd (HTTP Methods)
The BackEnd doesn't just give you data; it performs specific actions based on the "verb" you send:

| Method            | Action            | Example Scenario                                  |
| :---              | :---              | :---                                              |
| **GET**           | **Retrieve**      | Clicking the link to *view* the discount page.    |
| **POST**          | **Create**        | Filling out a form to *sign up* for the discount. |
| **PUT**           | **Update**        | Changing your profile email address.              |
| **DELETE**        | **Remove**        | Deleting a saved coupon from your account.        |

---

## 5. What the BackEnd Actually Does
Once the request hits the server, the BackEnd "Brain" takes over:
* **Business Logic:** It checks if the discount is still valid.
* **Database Interaction:** It looks up your user ID to see if you’ve used the code before.
* **API Calls:** It might talk to another service (like a payment processor) to verify the price.
* **Final Response:** It sends back the HTML/CSS/Data so the FrontEnd can show you the "Success!" page.
