Understanding the useRef Hook in React
In React, when a component's state changes, the entire component re-renders. This means all normal variables inside the function are recreated from scratch. If you need a value to stay the same even after the UI refreshes, or if you need to "touch" a HTML element directly, you use useRef.

1. The Problem with Normal Variables
When you use a standard variable like let a = 0; inside a component, it resets to 0 every time the component re-renders (e.g., when setCount is called). It cannot "persist" or remember its value across renders.
2. What is useRef?
useRef is a hook that returns a mutable object. This object has a single property called .current.

Persistence: The value stored in .current stays the same even when the component re-renders.

No Re-renders: Changing the value of ref.current does not trigger a UI re-render. This makes it different from useState.

JavaScript
const myRef = useRef(0); // returns { current: 0 }

3. Core Use Case: Accessing the DOM
The most common use for useRef is to grab a direct reference to a DOM element (like a button, input, or div) so you can manipulate it without using state.
Example: Changing a button's color from another button

Create the Ref: const btnRef = useRef(null);

Attach the Ref: Use the ref attribute on the element you want to control.

Manipulate the Ref: Access the element via btnRef.current.

JavaScript
import { useRef } from 'react';
function App() {
  const btnRef = useRef(null);

  const changeColor = () => {
    // Accessing the DOM element directly through .current
    btnRef.current.style.backgroundColor = "red";
  };

  return (
    <div className="p-4">
      {/* Target button linked via ref */}
      <button ref={btnRef} className="px-4 py-2 border">
        I will change color
      </button>

      {/* Trigger button */}
      <button 
        onClick={changeColor}
        className="ml-4 px-4 py-2 bg-blue-500 text-white"
      >
        Click to change other button's color
      </button>
    </div>
  );
}

Key Differences: useState vs useRef

Feature	                   useState	                                          useRef
Re-render	      Triggers a re-render when updated.	            Does not trigger a re-render.
Persistence	      Persists value across renders.	                Persists value across renders.
Usage	          Used for data that should show up on the UI.	    Used for DOM access or storing "hidden" variables.
Access	          Accessed directly (e.g., count).	                Accessed via .current (e.g., myRef.current).

Summary Checklist
Use useRef when you need to store a value that doesn't affect the UI layout.

Use useRef to focus an input, trigger an animation, or integrate with third-party DOM libraries.

Always access or modify the value using the .current property.
